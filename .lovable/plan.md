## Goal

New showcase page at `/showcase/elfas-namuose` for the brand **Elfas Namuose**, mirroring the existing MAKARA WEAR / Brite layout, populated with the most recent 58 generated visuals of the *Elves Behavin' Badly Jigsaw Puzzle* product from `info@tsimkus.lt`.

## What I found

- Pattern to follow: `src/pages/showcase/MakaraWearShowcase.tsx` (sibling: `BriteShowcase.tsx`, `BrandSampleShowcase.tsx`) — a single self-contained page using `LandingNav` + `LandingFooter` + `SEOHead` + a hardcoded `IMAGES` array + a stat strip + a 2/3/4-col image grid with lightbox.
- Routes registered in `src/App.tsx` via lazy import + `<Route path="/showcase/...">`.
- Source data: 58 completed `generation_jobs` rows for the puzzle (user `fe45fd27-...`, product `Elves Behavin' Badly Jigsaw Puzzle`, all created on 2026-05-13 between 14:34–14:35). Each has `scene_name` + a single image URL in `results[0]`. All scenes are still-life / object scenes — no people — so no Editorial vs Essentials split is meaningful here. I'll show them as a single flat grid (no category badge / filter).

## Implementation steps

1. **Create `src/pages/showcase/ElfasNamuoseShowcase.tsx`** — copy MAKARA structure exactly (same SEO `noindex`, same `LandingNav`/`LandingFooter`, same hero typography, same stat strip, same grid + lightbox component inline). Differences:
   - Eyebrow: `Brand example · ELFAS NAMUOSE`
   - H1: `This is what VOVV.AI makes from one product photo`
   - Subtitle: `A complete product visual library for Elves Behavin' Badly Jigsaw Puzzle — 58 visuals, ready for web, social, and listings`
   - SEO title: `A Sample Visual Library for Elfas Namuose | VOVV.AI`
   - SEO description: `A product editorial drop built for Elfas Namuose — generated by VOVV.AI from a single product photo.`
   - Stats: `{ value: '58', label: 'Visuals' }` and `{ value: '~3 min', label: 'Made in' }`
   - `IMAGES` array: 58 entries `{ url, scene }` from the query result, ordered by `created_at`. Drop the `category` field (not needed for this brand).
   - Aspect ratio: keep `aspect-[4/5]` to match the family.

2. **Register route** in `src/App.tsx`:
   - Add `const ElfasNamuoseShowcase = lazy(() => import('@/pages/showcase/ElfasNamuoseShowcase'));`
   - Add `<Route path="/showcase/elfas-namuose" element={<ElfasNamuoseShowcase />} />` next to the other three showcase routes.

3. **No backend / DB changes.** Page is `noindex` like the other showcase samples, so no sitemap update.

## Out of scope

- No CMS-driven approach (the existing showcases are hardcoded — staying consistent).
- No filtering UI / no scene-grouping tabs (puzzle scenes are all object stills, flat grid is the right call).
- No memory updates needed.

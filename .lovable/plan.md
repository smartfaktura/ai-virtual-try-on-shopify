

# Fix Short Film Audio: End-to-End Sound Production

## What's Actually Wrong

After thorough investigation, here are the findings:

**The ElevenLabs edge functions (music, sfx, tts) have ZERO logs** — they have literally never been called. The ElevenLabs connector IS linked and the API key IS set. The edge function code looks correct. So the problem is upstream: audio generation never triggers.

### Root Causes

1. **Default `audioMode` is `'silent'`** — users must manually switch to Music/Voiceover/Full Mix in Step 5 (Settings). Most users never change this, so audio generation is skipped entirely at line 913: `if (settings.audioMode !== 'silent' && settings.audioMode !== 'ambient')`.

2. **No script editing UI** — `script_line` values exist as defaults in `shortFilmPlanner.ts` and are generated by the AI planner, but there's no UI for users to see/edit them in the shot plan step. Users have no idea voiceover narration exists.

3. **Audio generation only runs AFTER video generation succeeds** — it's fire-and-forget at line 914. If the user closes the page during the long video generation wait, audio never runs. There's no way to trigger it independently on reload.

4. **Music and SFX are NOT synchronized to shot transitions** — music is generated as a single track for the total duration. SFX are generated per-shot with matching duration, but there's no crossfade or transition alignment. The `ShortFilmVideoPlayer` plays them simultaneously but doesn't time SFX to shot boundaries.

## Implementation Plan

### 1. Change default `audioMode` to `'full_mix'`
**File: `src/hooks/useShortFilmProject.ts`**
- Change `DEFAULT_SETTINGS.audioMode` from `'silent'` to `'full_mix'` so every short film gets music + SFX + voiceover by default

### 2. Add Script Line editing to Shot Plan UI
**File: `src/components/app/video/short-film/ShotPlanEditor.tsx`**
- Show each shot's `script_line` as an editable text input under the shot card
- Label: "🎙 Voiceover line" with a small text area
- When user edits, call `updateShot` to persist the change
- Show character count (target: 5-15 words)

### 3. Add standalone "Generate Audio" button on draft restore
**File: `src/hooks/useShortFilmProject.ts`**
- In `loadDraft`, after restoring shots, check if audio URLs are missing but `audioMode` requires them
- If so, set `audioPhase` to `'idle'` so the UI shows the Generate Audio button
- This allows audio generation independent of video generation

### 4. Time-align SFX to shot boundaries in the player
**File: `src/components/app/video/short-film/ShortFilmVideoPlayer.tsx`**
- Calculate cumulative start times for each shot based on `duration_sec`
- When playing SFX tracks, delay each shot's SFX to start at its calculated offset
- Same for voiceover tracks — start each at the shot's time offset
- Use `setTimeout` or `AudioContext` scheduling for precise timing

### 5. Add music prompt auto-generation from film context
**File: `src/hooks/useShortFilmProject.ts`**
- When `audioMode` is set to `music` or `full_mix` and no custom `musicPrompt` is set, auto-build a contextual prompt from `filmType` + `tone` + shot roles
- Example: "cinematic luxury ambient background music for a product launch film, slow building tension with elegant resolution"

# Plan — SEO pilot for `/discover/{slug}` (10 pages)

Upgrade the existing public `/discover/:itemId` route to behave like Pexels: in-app navigation keeps the fast modal, but direct hits, refreshes, and search crawlers get a fully rendered SEO page at the same URL. Ship 10 hand-picked presets first, prerender them, validate Search Console indexing, then scale to 30 → 100+.

Zero changes to `/app/discover` (private, behind auth). All content sourced strictly from existing `discover_presets` columns — no fabricated copy.

---

## 1. URL structure (confirmed)

- Public URL = `/discover/{slug}` using the human-readable slug already generated by the DB trigger `generate_discover_preset_slug()` (e.g. `/discover/structured-tote-on-marble-3a7c91`).
- React Router param stays `:itemId` internally (no route signature change), but the value passed in the URL is always the slug. `getItemSlug()` in `src/lib/slugUtils.ts` already returns the DB slug for presets.
- Raw UUIDs will never appear in any sitemap entry, anchor tag, or canonical.

---

## 2. SEO page content (real fields only, no thin pages)

Every prerendered page renders, in this order:

1. **Breadcrumbs** — `Home › Discover › {Category} › {Title}` (JSON-LD `BreadcrumbList` + visible nav).
2. **H1** — `{preset.title}`
3. **Hero image** — full-bleed `<img>` from `image_url`, alt = `${title} — AI ${category} visual`, `loading="eager"` for hero, `decoding="async"`.
4. **Metadata chips row** — category, subcategory, model_name, scene_name, aspect_ratio, quality (only render chips for non-null fields).
5. **About this visual** — `prompt` rendered as readable paragraph(s).
6. **Tags** — `tags[]` as chips, each linking to `/discover?tag={tag}` (internal linking).
7. **Related visuals (8 items)** — same `category` + matching `discover_categories`, excluding self. Reuses the existing related-items computation already in `PublicDiscover.tsx`. Each card links to `/discover/{relatedSlug}`.
8. **CTA** — "Open in Visual Studio" → `/auth?redirect=/app/discover/{slug}` (anonymous) or `/app/discover/{slug}` (logged in).
9. **Footer** — existing `PageLayout` footer (more internal links).

No invented descriptions, benefits, or claims. If a field is null, that section is omitted, not faked.

---

## 3. Meta description (safe template)

```
{title} — AI-generated {category} product visual created with VOVV.AI.
Explore the visual style, prompt, tags, and related product photography examples.
```

Uses only `title` + `category` (both NOT NULL in `discover_presets`). No prompt-text bleed. Truncated only if final string exceeds 160 chars (rare; safe trim at word boundary).

---

## 4. Canonical (single source of truth)

- Every page (modal open or closed) emits exactly: `<link rel="canonical" href="https://vovv.ai/discover/{slug}">`
- The modal does NOT change the URL to a query string or hash — it stays at `/discover/{slug}`, so the canonical is identical whether opened via modal or direct hit. No duplicate-canonical risk.
- The grid index `/discover` keeps its own canonical `https://vovv.ai/discover`.

## 5. `/app/discover` safety (confirmed, multi-layer)

| Guard | Mechanism |
|---|---|
| Not in sitemap | `MARKETING_URLS` allowlist contains no `/app/*`; new build assertion throws if any emitted `loc` starts with `/app/` |

| UI / behavior | Zero edits to `Discover.tsx` other than the one-line `<SEOHead noindex />` injection |

## 6. Modal behavior (Pexels-style, careful)

Detection logic in `PublicDiscover.tsx`:

```text
urlItemId present + cameFromGridClick (state)  → modal open over grid (current behavior)
urlItemId present + no internal nav state      → render full SEO detail view (no modal flash)
no urlItemId                                    → grid only (current behavior)
```

Implementation:
- Grid card click: `navigate('/discover/' + slug, { state: { fromGrid: true } })` then `setSelectedItem(item)` → modal opens, no page reload.
- Browser back from modal: `navigate(-1)` from modal close → returns to `/discover` grid, modal closes, scroll position restored.
- Direct URL / refresh / Googlebot: `location.state?.fromGrid` is undefined → render full SEO view, no modal.
- Closing modal on a direct-loaded SEO page: stays on the SEO page (no jarring navigation to grid).

This preserves all current SPA UX and adds the SEO render path strictly as an additive branch.

---

## 7. Sitemap safety

`scripts/generate-sitemap.ts` changes:

1. Add `PILOT_DISCOVER_SLUGS: string[]` constant (10 hardcoded slugs picked via SQL — see Pilot selection below).
2. Before emitting, validate each slug against `discover_presets.slug` via a one-off Supabase fetch (script already runs in Node; uses anon key + RLS public read). Missing slug → build fails with explicit error.
3. **Hard assertion** after the dedupe step:
   ```text
   for (const e of sorted) {
     if (e.loc.startsWith('/app/')) throw new Error(`Forbidden /app/ URL in sitemap: ${e.loc}`);
   }
   ```
4. Each `/discover/{slug}` entry: `priority: 0.7`, `changefreq: monthly`, `lastmod: today`.

200-status validation happens automatically in `scripts/prerender.ts` (any non-200 → build fails, Layer A–D guards).

---

## 8. Prerender validation (post-build acceptance)

`scripts/validate-prerender.ts` (new helper, ~30 lines) loops the 10 pilot slugs and asserts for each:

- File `dist/discover/{slug}/index.html` exists and is > 5 KB
- HTML contains `<title>` matching `{preset.title}`
- HTML contains `<meta name="description"`
- HTML contains `<link rel="canonical" href="https://vovv.ai/discover/{slug}">`
- HTML contains the literal `image_url` from the preset
- HTML contains `<script type="application/ld+json">` with both `ImageObject` and `BreadcrumbList`
- HTML contains an `<h1>` with the preset title text
- HTML does NOT match `<div id="root"></div>\s*</body>` (empty SPA shell guard)

Wired into `npm run build` as final step. Any failure → build exits non-zero.

## 10. No fake content

## Pilot slug selection (one-off SQL before implementation)

```sql
SELECT slug, title, category
FROM discover_presets
WHERE is_featured = true
  AND image_url IS NOT NULL
  AND length(prompt) > 100
ORDER BY category, sort_order
LIMIT 10;
```

Distribute across discover families (Bags, Beauty, Fragrance, Footwear, Apparel, Eyewear, Activewear, Home, Accessories, Editorial). Resulting slugs pasted into `PILOT_DISCOVER_SLUGS`.
